# O tipo float guarda números de ponto-FLUTUANTE de dupla precisão, ...
# ... cuja faixa depende do compilador  C (ou C# ou Java) que foi ...
# ... usado para construir o Python; eles possuem uma precisão ... 
# ... limitada e não podem ser comparados com segurança por igualdade.
# Números do tipo flutuante são escritos com um ponto decimal, ...
# ... ou usando notação exponencial, por exemplo, ...
# ... 0.0, 4., 5.7, -2.5, -2e9, 8.9e-4

# A inexatidão não é um problema específico para o Python

# Se precisarmos realmente de altas precisões exatas, ... 
# ... os números do Python 'decimal.Decimal' do modulo ... 
# ... 'decimal' podem ser usados. A realização destes ...
# ... cálculos é precisa, no nível de precisão que ...
# ... especificamos (por padrão, há 28 casas decimais) ...
# ... e pode representar exatamente números periódicos, ...
# ... como 0.1; mas o processamento é bem mais lento ...
# ... do que com os 'floats'. Por causa de sua precisão ...
# ... números decimal.Decimal são apropriados para ...
# ... cálculo financeiro.

# O modo aritmético misturado é suportado, por exemplo, ...
# ... por um 'int' e um 'float' que produz um 'float' ...
# ... e um 'float' e um 'complex' produzem um 'complex'.
# Pelo fato de 'decimal.Decimal' possuir precisoes fixas, ...
# ... ele só pode ser um resultado 'decimal.Decimal'.
# Se uma tentativa de operacao usa itens incompatíveis
# uma exceção 'TypeError' é lançada.
